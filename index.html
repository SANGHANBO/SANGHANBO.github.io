Ubuntu下安装colmap

Veröffentlicht am 2020-02-20

本文主要参考 https://colmap.github.io/install.html
进行COLMAP的安装，安装环境为Ubuntu-14.04.1、gcc-4.8.4、cuda-9.0。

首先安装文档中Linux部分给出的依赖包。
由于ceres-solver要求cmake>=3.5，Ubuntu14下apt-get默认安装的版本是cmake-2.8，因此需要手动升级。下载cmake-3.5安装包：

若想保留原始cmake(以免破坏环境)，可以为本次安装指定目录：

此时默认cmake仍为旧版，在~/.bashrc中指定路径：

接下来按照教程安装完成ceres-solver，便可以开始着手colmap的安装了。
cmake过程中，注意一些依赖库可能被环境中的其它库覆盖，例如anaconda3/lib中的库导致的覆盖：

解决方法如下：
首先输入env查看系统环境，如果PATH中包含anaconda路径，则在~/.bashrc删除该路径：

sudo apt-get remove qtbase5-dev

wget http://download.qt.io/archive/qt/5.4/5.4.0/qt-opensource-linux-x64-5.4.0.run

cmake Qt5_DIR=$QT5-DIR/Qt5.4.0/5.4/gcc_64/lib/cmake/Qt5 ..

export Qt5_DIR=$QT5-DIR/Qt5.4.0/5.4/gcc_64/lib/cmake/Qt5
cmake ..
make -j64
sudo make install

colmap: symbol lookup error: colmap: undefined symbol: _ZTI13QOpenGLWidget

export LD_LIBRARY_PATH=$QT5-DIR/Qt5.4.0/5.4/gcc_64/lib/:$LD_LIBRARY_PATH


大功告成！
